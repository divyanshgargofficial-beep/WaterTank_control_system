# Render deployment

This backend is deployed as a Render Docker web service with Render Postgres.

## Service

- Name: `water-tank-cloud-backend`
- Runtime: Docker
- Plan: Free
- Public URL: `https://water-tank-cloud-backend.onrender.com`
- Health endpoint: `https://water-tank-cloud-backend.onrender.com/health`
- Database health endpoint: `https://water-tank-cloud-backend.onrender.com/health/db`
- Production command: `npx prisma migrate deploy && node dist/server.js`

The root `render.yaml` is the primary Blueprint file. It creates the web service and the `water-tank-controller-db` Postgres database.

## Required environment variables

Render should provide these through the Blueprint or dashboard. Do not commit real secret values.

- `NODE_ENV=production`
- `PORT=8080`
- `DATABASE_URL` from Render Postgres
- `JWT_SECRET` generated by Render
- `JWT_EXPIRES_IN=12h`
- `DEVICE_TOKEN` must match `deviceToken` in Package 1 firmware
- `DEFAULT_DEVICE_ID=package-1`
- `DEFAULT_DEVICE_NAME=Home Water Tank`
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`
- `FAMILY_EMAIL`
- `FAMILY_PASSWORD`
- `CORS_ORIGIN=*`

## Prisma

The Docker startup command runs:

```sh
npx prisma migrate deploy
```

This applies the checked-in migrations from `prisma/migrations` to the Render Postgres database before the server starts.
